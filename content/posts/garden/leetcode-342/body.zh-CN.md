**核心思路**

`4^k` 本质上是更严格的 `2` 的幂。

先确认 `n` 只有一个 `1` 位：

```cpp
n > 0 && (n & (n - 1)) == 0
```

做到这一步后，剩下的问题只有一个：这个唯一的 `1` 是否落在正确的位置上。

**方法 1：`mod 3`**

- 因为 `4 ≡ 1 (mod 3)`，所以 `4^k - 1 ≡ 0 (mod 3)`。
- 因此在已经确认是 `2` 的幂的前提下，满足 `(n - 1) % 3 == 0` 的就是 `4` 的幂。

```cpp
class Solution {
public:
    bool isPowerOfFour(int n) {
        return n > 0 && (n & (n - 1)) == 0 && (n - 1) % 3 == 0;
    }
};
```

**方法 2：检查位位置**

- `4` 的幂的二进制形态是 `1`、`100`、`10000`、`1000000` ...
- 如果按从 `0` 开始编号，唯一的 `1` 总是落在 `0、2、4...` 这些位置。
- 在 32 位整数里，`0xAAAAAAAA` 的二进制是 `10101010 10101010 10101010 10101010`，它标记的是所有奇数编号位。
- 如果 `n` 已经被确认是 `2` 的幂，那么 `(n & 0xAAAAAAAA) == 0` 就说明那个唯一的 `1` 不在奇数编号位上。

```cpp
class Solution {
public:
    bool isPowerOfFour(int n) {
        return n > 0 && (n & (n - 1)) == 0 && (n & 0xAAAAAAAA) == 0;
    }
};
```

**实践备注**

- 可读性上，`mod 3` 版本通常更直接。
- 掩码版本是纯位运算技巧，但掩码依赖整数位宽。若是 64 位整数，应改为 `0xAAAAAAAAAAAAAAAAULL`。
- “奇数位 / 偶数位”的表述取决于编号方式：从 `0` 开始编号时，`1` 在偶数位；从 `1` 开始编号时，`1` 在奇数位。
