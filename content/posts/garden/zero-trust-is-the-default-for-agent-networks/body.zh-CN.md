我反复回到的一个直觉是：如果智能体系统真的变成一个网络——发布/订阅、P2P/A2A 消息传递、任务队列、工具分发，甚至智能体雇佣人类——那么 **零信任** 就不再只是一个安全口号，而会成为基础架构的默认前提。

每一条边界都有风险：一个智能体调用另一个智能体、调用工具、接触私有数据，或转移资金。我会假设没有任何节点是被完全信任的；它只能被验证、被约束、被观测。在实践中，这意味着：

- **去中心化凭证与证明**，而不是由某一个平台充当通用的信任根。
- **面向智能体与工具的可信沙箱执行**：临时状态、彻底清理、严格的出口控制，以及 <u>围绕敏感数据建立清晰边界</u> 以限制泄漏。
- **双向防御**：既保护企业免受智能体的影响，也保护智能体免受恶意工具、提示、对等节点或被攻陷运行时的侵害。
- **具备企业级可审计性的可观测性**，同时不把整个系统变成永久性监控。
- **机器原生支付通道**，让经济交换成为一等能力，而不是勉强套进面向人类的结账流程。

权威并不会消失；它会变成证明来源和合规锚点，而不是整个信任模型本身。

经济层的重要性比最初看上去更高。开放协议往往会把价值向网络外推。这意味着，变现方式会从拥有整个产品，转向拥有链条中的某个可信节点：身份、执行、声誉、合规、结算。

这也是为什么区块链会不断在智能体讨论中反复出现——不是出于意识形态，而是作为基础设施。Molthunt 是一个有用的信号：项目在 Base 上铸造代币，使用 x402 风格的支付，而智能体投票本质上就是链上签名。一旦交互携带签名、激励与转移，它们就不再只是 API 调用了，而是经济事件。
