**CRC 是什么**
CRC 是一种广泛用于数字网络和存储设备的差错检测码。发送方把比特流视为 `GF(2)` 上的多项式，用固定的生成多项式做除法，并把余数附加为简短的校验值。

**为什么它本质上只是异或**
CRC 使用模 2 算术：
- `1 - 1 = 0`
- `1 - 0 = 1`
- `0 - 1 = 1`
- 减法等价于异或
- 没有进位，也没有借位

**计算步骤**
- 设生成多项式 `G(x)` 的最高次幂为 `n`。
- 把 `G(x)` 的系数写成二进制；这个长度为 `n+1` 的值就是除数。
- 在原始消息后补 `n` 个 `0`。
- 用补零后的消息作为被除数，按高位对齐做模 2 除法。
- 最终得到的 `n` 位余数就是 CRC。
- 把这个余数附加到原始消息后，得到 CRC 码字。

**例子**
- 信息码字：`11100011`
- 生成多项式：`G(x) = x^5 + x^4 + x + 1`
- 对应二进制除数：`110011`
- 最高次幂是 `5`，所以先补 5 个 `0`：`1110001100000`
- `1110001100000` 用 `110011` 做模 2 除法，余数为 `11010`
- 发送码字：`11100011` + `11010` = `1110001111010`

**接收端验证**
接收方用同样的生成多项式再除一遍收到的码字：
- 余数为 `0` -> 未检测到错误
- 余数非 `0` -> 检测到传输或存储中的损坏

**使用边界**
CRC 非常适合以很低成本检测随机比特翻转，但它**不是**可靠的完整性或抗篡改机制：
- CRC 是线性的
- 攻击者可以修改数据后重新计算一个合法 CRC
- 如果威胁模型包含主动篡改，应使用密码学哈希，或更合适的 MAC
