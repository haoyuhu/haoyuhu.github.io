## 问题概述

打家劫舍 II 是经典打家劫舍问题的变体，房屋**环形排列**，意味着第一间和最后一间房屋是相邻的，不能同时被偷盗以避免警报触发。

给定数组 `nums` 表示每间房屋中的金额，计算在不触发警报的情况下，能够偷窃的最大金额。

---

## 关键思路：环形约束与状态编码

在线性房屋中，只需保证不偷相邻房屋即可。

但环形中，首尾相邻，若两端房屋都被偷则触发警报。

因此，需要额外状态记录**第一间房屋是否被偷**。

### 两位二进制状态编码

用两位二进制数编码状态：

| 状态位 | 含义                              | 说明                                    |
|--------|----------------------------------|-----------------------------------------|
| 高位   | 第一间房屋是否被偷                | 0 = 未偷，1 = 已偷                      |
| 低位   | 当前房屋（第 i 家）是否被偷       | 0 = 未偷，1 = 已偷                      |

共四种状态：

| 二进制 | 状态说明                             |
|--------|------------------------------------|
| `00`   | 第一间未偷，当前未偷                 |
| `01`   | 第一间未偷，当前被偷                 |
| `10`   | 第一间被偷，当前未偷                 |
| `11`   | 第一间被偷，当前被偷                 |


### 状态转移方程

令 `robMoney[i][state]` 表示偷到第 i 家时对应状态下的最大金额：

```cpp
robMoney[i][0] = max(robMoney[i-1][0], robMoney[i-1][1]);
robMoney[i][1] = robMoney[i-1][0] + nums[i];
robMoney[i][2] = max(robMoney[i-1][2], robMoney[i-1][3]);
robMoney[i][3] = robMoney[i-1][2] + nums[i];
```

- 状态 `0, 1` 对应第一家未被偷的情况；
- 状态 `2, 3` 对应第一家已被偷的情况。


## 注意点

- 在最后一家（索引 `n-1`），状态 `11`（首尾均被偷）非法，应排除。
- 特殊情况：仅有一间房时，`11` 状态有效，即可偷该间。
- 第一间房（索引 `0`）状态 `01` 和 `10` 逻辑冲突，赋值为最小值 `MIN_INT` 以无效化。

## AC代码示例

```cpp
class Solution {
public:
    static constexpr int MIN_INT = -999999999;

    int max(int x, int y) { return x > y ? x : y; }

    int max(int x, int y, int z) {
        return max(x, max(y, z));
    }

    int rob(const std::vector<int>& nums) {
        int n = nums.size();
        if (n == 0) return 0;
        if (n == 1) return nums[0];

        std::vector<std::vector<int>> robMoney(n, std::vector<int>(4, 0));

        robMoney[0][3] = nums[0];       // 第1家被偷，当前也是第1家
        robMoney[0][1] = robMoney[0][2] = MIN_INT; // 无效状态

        for (int i = 1; i < n; ++i) {
            robMoney[i][0] = max(robMoney[i-1][0], robMoney[i-1][1]);
            robMoney[i][1] = robMoney[i-1][0] + nums[i];
            robMoney[i][2] = max(robMoney[i-1][2], robMoney[i-1][3]);
            robMoney[i][3] = robMoney[i-1][2] + nums[i];
        }

        // 最后一家不能是 11 状态
        return max(robMoney[n-1][0], robMoney[n-1][1], robMoney[n-1][2]);
    }
};
```

## 方案总结

| 方面               | 细节说明                                  |
|--------------------|-------------------------------------------|
| 问题               | 环形房屋不允许邻居同时被偷               |
| 核心挑战           | 首尾相邻，不能同时偷                       |
| 状态表示           | 两位二进制，[第一家是否偷][当前家是否偷] |
| 状态转移           | 根据前状态转移，禁止非法状态              |
| 边界处理           | 单家情况特殊处理，首尾状态检查            |

此方法通过状态扩展，巧妙解决环形依赖关系，实现了 O(n) 的动态规划解法。

---

*注：* 该方法实际相当于同时考虑偷和不偷第一家的两条路径，使用状态压缩统一处理，提高代码整洁度和效率。
