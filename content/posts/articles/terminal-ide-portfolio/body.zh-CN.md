# 为什么我的作品集要长得像一台 IDE

很多作品集首页都在做同一件事：把信息铺平、把经历压缩、把表达变成几个通用卡片。

我更喜欢把个人站做成一台可以被阅读的“工作环境”。

## IDE 风格适合展示什么

- 它天然支持“文件与模块”的心智模型。
- 它允许内容带着上下文一起出现，而不是孤立地摆在页面上。
- 它能把经历、项目、短文、长文和本地创作工具放到同一个叙事空间里。

## 为什么这次要做配置驱动

过去很多个人站会把个人信息直接写在前端组件里。短期很快，长期会让内容更新越来越难。

这次重构的目标是：

1. 所有可维护内容都进入 YAML / Markdown。
2. 页面只渲染 bundle，不直接持有个人信息。
3. 本地 CLI 与 Studio GUI 负责导入、整理、预览与发布。
4. 发布前自动跑校验、类型检查和浏览器回归。

## 结果会是什么

作品集不再只是一个“展示结果”的页面，而是一个“生成结果”的系统。
