## 问题概述
题目给出一个 MxN 的房间二维网格，每个格子包含一个整数：
- 负数表示进入该房间会扣血（被恶魔攻击）。
- 0 表示空房间。
- 正数表示回血道具。

骑士从左上角(0,0)出发，必须只向右或向下移动，目标是救出被关在右下角(M-1,N-1)的公主。骑士的生命值降至0或以下即死亡。

任务是**计算骑士能够安全抵达公主处所需的最小初始生命值。**

### 约束
- 初始生命值为正整数，最大值无上限。
- 房间内的整数可以为任意值，包括起点和终点。

---

## 核心思路：动态规划状态定义

### 正向DP问题分析
尝试定义 `dp[i][j]` 为从起点安全抵达 `(i,j)` 所需的最小初始生命值。

但实际情况中，路径上的剩余生命影响后续走法，单纯取两个方向的较小值并不能保证最终安全通关。

原因在于，较低的初始需求可能陪伴低剩余生命，中途遇到负房间就会死亡。

### 反向DP解决方案
定义 `dp[i][j]` 为骑士进入房间 `(i,j)` 时**必须具备的最小生命值**，才能保证后续安全到达终点。

转移方程为：

```cpp
// 状态转移：
dp[i][j] = max( min(dp[i+1][j], dp[i][j+1]) - dungeon[i][j], 0 );
```

- `min(dp[i+1][j], dp[i][j+1])` 表示下一步需要的最小生命值。
- 减去当前房间的增减量后，若结果小于等于0则至少需要1血生命值进入。

边界上，假设越界房间需求为无穷大，公主房间的需求为0，然后递归计算。

`
最终 `dp[0][0] + 1` 即为骑士最小初始生命值。

## 代码实现（C++）

```cpp
class Solution {
public:
    int calculateMinimumHP(vector<vector<int>>& dungeon) {
        int rows = dungeon.size();
        int cols = dungeon[0].size();
        
        vector<vector<int>> dp(rows + 1, vector<int>(cols + 1, INT_MAX));

        dp[rows][cols - 1] = 1; // 进入公主房间前至少1血
        dp[rows - 1][cols] = 1;

        for (int i = rows - 1; i >= 0; --i) {
            for (int j = cols - 1; j >= 0; --j) {
                int need = min(dp[i + 1][j], dp[i][j + 1]) - dungeon[i][j];
                dp[i][j] = (need <= 0) ? 1 : need;
            }
        }

        return dp[0][0];
    }
};
```

## 小结
- 本题关键在于从终点向起点反向定义状态，使状态含义明确，避免按路径考虑剩余血量复杂度。
- 动态规划的转移定义为进入当前房间所需最小生命值，递推有效且易于实现。
- 时间复杂度 O(M*N)，空间复杂度 O(M*N)，符合题目性能要求。

这种反向思路值得学习并应用于多路径状态受限的动态规划问题中。
