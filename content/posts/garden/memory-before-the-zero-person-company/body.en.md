**Working note**

The interesting frontier is not just better **autoresearch**. It is whether agents can carry memory across surfaces, share context with a team, and stay bounded enough to be trustworthy.

I keep coming back to three layers of memory:
- **Personal memory**: preferences, recurring tasks, unfinished threads.
- **Task memory**: what this investigation already tried, ruled out, or learned.
- **Team memory**: shared facts, decisions, and patterns that should outlive one chat window.

Without this stack, multi-end agent workflows still feel like demoware. An agent that works on desktop but forgets everything on mobile is not a teammate; it is a session.

This matters even more for dense research domains. A good autoresearch system should be able to mine sources like **Seeking Alpha**, compare theses, and run source-vs-source “duels.” But the real leverage comes when that research is retained, linked, and reusable.

I am also increasingly convinced that the **copilot agent** should be separated from the more autonomous mesh. In an agent organization, each agent needs a crisp role definition and an explicit guard layer: what it can read, write, trigger, and escalate. The **zero-person company (ZPC)** idea is exciting, but the operational prerequisite is much less glamorous: memory, permissions, and handoff design.

My current heuristic is simple: build the **copilot** first, then the **agent team**. If the personal assistant cannot remember and coordinate safely across endpoints, scaling to a zero-human company is mostly theater.
