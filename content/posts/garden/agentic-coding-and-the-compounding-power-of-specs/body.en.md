My current thesis is that **agentic coding is a key unlocking mechanism for general intelligence**. Not because coding is uniquely sacred, but because software work sits at the junction of language, logic, tools, memory, and execution. Once agents can operate there reliably, they do more than write code: they begin to rewire the surrounding organization.

The useful mental model is this: agentic systems keep lowering the amount of raw effort required to move work forward. First they remove obvious manual steps. Then they start compressing tasks that still need some attention but very little judgment. Eventually, even low-cognition participation starts to look like physical labor—something the system should absorb, not something a person should spend time on.

That inversion matters. As low-judgment work gets pushed into the substrate, the value of high-judgment work goes up: framing the problem, writing a precise spec, choosing tradeoffs, defining failure modes, and reviewing outcomes.

This is why I keep returning to **specs**. A spec is not just documentation for an agent. It is the executable boundary between human intent and machine action. When written well, it encodes assumptions, standards, interfaces, and edge cases in a reusable form.

Once that happens, specs start to compound:

- they make delegation cheaper,
- they turn organizational knowledge into durable assets,
- they restructure workflows around intent, review, and exception handling.

So the endgame is not “AI helps engineers code faster.” It is **organizational engineering with compounding returns**, powered by specs. That feels like the deeper change.
